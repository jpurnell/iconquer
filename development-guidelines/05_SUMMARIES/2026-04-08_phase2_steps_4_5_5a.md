## 5. Key Decisions Locked In

1. **`hash()` is FNV-1a 64-bit, no dependencies.** The use case (replay-staleness detection) doesn't need cryptographic strength. Zero new deps was a hard constraint per the project's blast-radius rule.

2. **`legalMoves` for `placeArmies` is canonical** — one move per owned country with `count = unallocatedArmies`, not the cross product. LLMs and AI agents that want to split allocation across countries call repeatedly.

3. **`MatchResult` was added** beyond what `MultiAgentPlayerBinding.md` §3 sketched. The proposal sketched `matchDidEnd(result:)` referencing a `MatchResult` type but didn't define it. Defined as `(winnerId, totalTurns, totalMoves, finalStateHash)` — minimal, codable, replay-friendly.

4. **`MoveRecord.reasoning` stays inline.** Phase 3 LLM-tournament strategy-doc fuel. Bounded by `MatchSettings.maxReasoningLength`; longer entries truncated with explicit `reasoningTruncated: Bool`.

5. **`SeatBinding` is `Sendable` but not `Codable`** — it holds an `any PlayerAgent` existential which can't round-trip through JSON. Bindings are constructed at runtime; only `MoveRecord`/`MatchResult` need to serialize.

### Immediate Starting Point

**Step 6 — `MatchRunner` actor.** The big piece of `IconquerMatch`. Per `MultiAgentPlayerBinding.md` §3:

```swift
public actor MatchRunner {
    public init(
        initialState: GameState,  // → Game
        bindings: [SeatBinding],
        settings: MatchSettings,
        clock: any Clock<Duration> = ContinuousClock()
    ) throws

    public func run() -> AsyncThrowingStream<MoveRecord, Error>
    public func snapshot() -> GameState  // → GameSnapshot
}
```

**Sub-tasks (RED → GREEN, one at a time):**

1. **DESIGN:** Confirm the actor's contract: holds a `Game` value (mutating internally), validates `[SeatBinding]` covers exactly the players in `Game.players`, drives the turn loop until `phase == .victory` or fatal abort.
2. **RED:** Test runner with 2 deterministic stub agents on a 2-country game. Assert the stream yields N `MoveRecord`s ending in victory.
3. **GREEN:** Implement `MatchRunner.run()` driving a basic loop:
   - Capture `stateHashBefore` via `Game.snapshot().hash()`
   - Look up the binding for `game.currentPlayerId` (or pendingInput's player)
   - Call `agent.requestMove(state:seat:deadline:)` with `deadline = clock.now + binding.timeout`
   - Validate the returned move against `Game.legalMoves(for:)` — reject illegal
   - On success: `game.apply(move)`, build a `MoveRecord`, yield it
   - On `Game.phase == .victory`, build a `MatchResult` and end the stream
4. **RED:** Test with mixed agents (1 scripted + 1 stub MCP-style). Verify both seats get called.
5. **RED:** Timeout test using a `SlowAgent` that sleeps past its deadline. Verify `AgentError.timeout` is caught and `FallbackPolicy.randomLegalMove` produces a legal move with `MoveRecord.fallbackUsed == true`.
6. **GREEN:** Per-seat timeout enforcement via `withTimeout` (or `Task` + `clock.sleep(until:)` race). Fallback dispatch.
7. **RED:** Illegal move test with a `CheatingAgent` returning a move not in `legalMoves`. Verify rejection + fallback.
8. **GREEN:** Move validation in the dispatcher.
9. **RED:** State-hash staleness test. Submit a move whose `stateHashBefore` doesn't match the runner's hash. Verify rejection.
10. **GREEN:** Hash check.
11. **RED:** Replay test — feed a recorded `[MoveRecord]` log into a stub-agent runner that returns the logged move; final `stateHash` must equal the original.
12. **GREEN:** Replay support.

### Implementation Notes

- **`withTimeout`:** Swift 6 has `Task.timeout` patterns via `TaskGroup`. Cleanest is to race `agent.requestMove(...)` against `Task { try await clock.sleep(until: deadline); throw AgentError.timeout(seat:) }` in a `withThrowingTaskGroup`. Cancellation propagates to the agent task naturally.
- **`Game.legalMoves(for:)` validation:** the `move == .attack(...)` and `move == .placeArmies(...)` cases have free parameters that the canonical enumeration might not exactly match. For attack: any `(from, to)` pair where `canAttack(from, to)` is fine, regardless of mode. For placeArmies: any owned country with `count > 0` and `count <= unallocatedArmies` is fine. Loosen the canonical comparison appropriately or use an explicit `isLegal` predicate instead of `legalMoves.contains`.
- **`fallback: .randomLegalMove(seed:)`:** seeded `SeededRNG` (already in IconquerCore as `public`), pick uniformly from `Game.legalMoves(for: currentPlayerId)`.
- **`Game.snapshot()` is non-mutating** but `MatchRunner` holds the `Game` as `var` because `Game.apply(_:)` is mutating. The actor isolation makes that fine.
- **MockMCPAgent in tests:** trivial stub that returns scripted moves — same pattern as `StubAgent` in `CoreTypesTests.swift` but with a queue.

### Critical Context-Loss Warnings

1. **`legalMoves` returns canonical moves only.** When `MatchRunner` validates an agent's move, do NOT use `legalMoves.contains(move)` directly — `placeArmies(c, count: 3)` won't match `placeArmies(c, count: 5)` even though both might be legal. Use a separate `Game.isLegal(_:)` predicate or relax the comparison. **Decide this in Step 6 before writing the validator.**

2. **`MatchResult` was added beyond what the proposal sketched.** Don't be confused if you re-read `MultiAgentPlayerBinding.md` and don't see it defined — that's the proposal predating the integration.

3. **`hash()` is non-cryptographic.** If a future change tempts you to "upgrade to SHA256," the dependency-free constraint stays. The use case is staleness detection, not adversarial defence.

4. **Step 3's `PlayerStrategy` is still SUPERSEDED.** When you eventually do Step 5b (reshape `RandomStrategy` → `RandomAgent`), delete `PlayerStrategy.swift` outright. Don't keep it as a deprecated shim.

5. **`PlayerAgent` is Sendable + protocol-with-Sendable-existential.** `SeatBinding.agent: any PlayerAgent` works because the protocol requires `Sendable`. Don't add a non-Sendable conformance.

6. **`Duration` is Codable on macOS 13+ / iOS 16+.** OS 26 platforms guarantee this. `MoveRecord.latency` and `MatchSettings.*Timeout` rely on it.

7. **`IconquerMatch` does NOT depend on `IconquerAI`.** The dep graph is `IconquerCore` → `IconquerMatch` → `IconquerAI`. Random/Greedy will conform to `PlayerAgent` from inside `IconquerAI`, which depends on Match — never the other way around.
