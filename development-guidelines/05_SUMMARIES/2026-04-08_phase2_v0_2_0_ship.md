## IconquerCLI surface

- **Subcommands:** `simulate`, `play`, `replay`, `tournament`, `config`
- **Settings:** XDG-compliant (`$XDG_CONFIG_HOME/iconquer-cli/settings.json`)
- **Renderer:** ANSI with `--width`, `--ascii`, `--no-color`, `NO_COLOR`
  env support, `TIOCGWINSZ` auto-detect
- **Command parser:** in-REPL grammar with case-insensitive aliases
- **Agents:** `random`, `greedy` via `AgentFactory`
- **Maps:** `duel`, `line4` via `StarterMaps`
- **Executable:** `iconquer-cli` via swift-argument-parser

61/61 tests passing across 10 suites. Zero warnings.

## Notable design decisions

- `PlayRunner` is synchronous turn-based for v0.1.0 (NOT MatchRunner-driven)
  to avoid the top-level-async + `exit()` complications hit on iconquer-mcp.
- `@main` + top-level-code conflict resolved by renaming `main.swift` →
  `IconquerCLICommand.swift` (same trick used in IconquerMCP).
- `PlayRunner.runGame` takes a `Settings` parameter so tests can pass
  `Settings(assignCountries: false)` to start in `pickCountries` phase.
- Replay tests pin player names to `"Player 1"` / `"Player 2"` to match
  SimulateRunner exactly — Player.name is part of the snapshot hash.
